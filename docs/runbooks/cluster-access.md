# Runbook: Cluster Access

**Target System:** Kubernetes Cluster
**Author(s):** Riccardo Cereghino
**Last Updated:** 2026-03-14

## Objective
Gain authenticated access to the Kubernetes control plane and Talos API.

## Prerequisites
* `tofu` CLI v1.6.0+ installed
* `talosctl` CLI installed
* `kubectl` CLI installed
* SOPS age key at `~/.config/sops/age/keys.txt`
* `secrets/local.env.yaml` file available

## Execution Steps

### 1. Environment Setup
Load SOPS-encrypted secrets into your shell:
```bash
source scripts/env.sh
```
This decrypts `secrets/local.env.yaml` using your age key.

### 2. Retrieving Kubeconfig from OpenTofu
The kubeconfig is generated by OpenTofu and uses client certificate authentication.
```bash
cd infrastructure
tofu output -raw kubeconfig > ~/.kube/config
chmod 600 ~/.kube/config
```

### 3. (Alternative) Generating Kubeconfig from Talos
Alternatively, generate a fresh kubeconfig directly from Talos:
```bash
talosctl kubeconfig --nodes <control-plane-ip>
```

## Validation
Verify access to both the Kubernetes and Talos APIs:
* Run `kubectl get nodes` and verify the output contains a list of nodes showing `Ready` status.
  ```bash
  # Kubernetes API
  kubectl get nodes
  ```
* Run `talosctl health` and verify the output contains a healthy cluster response.
  ```bash
  # Talos API
  talosctl health --nodes <control-plane-ip>
  ```

## Troubleshooting
* **Error:** `decryption failed`
  * **Cause:** Wrong or missing age key
  * **Fix:** Check `~/.config/sops/age/keys.txt` is populated correctly.
* **Error:** `dial tcp 46.x.x.x:6443: i/o timeout`
  * **Cause:** Network layer blocking API access, often due to changing operator IPs.
  * **Fix:** Ensure Hetzner firewall rules allow your current IP address (reference Post-Mortem 001).
